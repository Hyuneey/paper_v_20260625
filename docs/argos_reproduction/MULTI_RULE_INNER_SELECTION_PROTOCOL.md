# Multi-Rule Inner Selection Protocol

Stage E2X-S uses only the inner-selection partition. Every panel rule is replayed twice in fresh rootless Podman containers with values-only mounts. Labels are loaded by trusted host metric code only after the panel and deterministic prediction hashes are frozen.

Direct binary predictions receive PA-free point and event diagnostics. Four arms are frozen per KPI: Best-1, Top-3 OR, Coverage-3 OR, and All-10 OR. Best-1 and Top-3 follow the declared metric ordering. Coverage-3 greedily adds rules by newly covered inner anomaly events and the predeclared tie order. No rule is repaired, removed for poor performance, or selected with outer data.
