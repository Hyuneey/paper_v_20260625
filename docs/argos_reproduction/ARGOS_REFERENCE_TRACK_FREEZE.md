# ARGOS Reference-Track Freeze

## Recommendation

`freeze_ARGOS_reference_track`

The core reference questions are answered: one-shot generation and isolated
runtime were verified; rule-only limitations, detector baselines, generic and
error-conditioned fusion, Repair recovery, Review inner and outer transfer,
and the complete A0-A3 branch comparison were measured.

Further prompt, model, branch, or detector tuning against the already exposed
outer partition has diminishing scientific value and would increase adaptive
overfitting risk. The primary thesis contribution now requires multivariate
SWaT evidence.

## Freeze Meaning

The freeze preserves all reports, code, selections, and private artifacts. It
does not delete or reinterpret the ARGOS track. After freeze:

- do not tune prompts or models against the exposed outer partition;
- do not introduce a new branch or branch selection;
- do not select LSTMADalpha or LSTMADbeta;
- do not attempt to improve the reported outer results;
- do not access the sealed test without explicit professor/researcher approval.

## Sealed-Test Alternatives

### Option A: Professor-Approved Joint Sealed Confirmation

Pre-register both detector variants and detector-only, A0, A1, A2, and A3.
Permit one execution with no branch winner selection, fallback, or policy
change. Interpret it as confirmation of component evidence, not exact ARGOS
replication.

### Option B: Preserve Sealed Test and Move to the Proposed Method

This is the recommended default when thesis time is limited. Existing outer
component evidence is adequate for design decisions, while official SWaT
evidence directly addresses the proposed multivariate contribution.

No choice here is based on unseen sealed-test performance.
